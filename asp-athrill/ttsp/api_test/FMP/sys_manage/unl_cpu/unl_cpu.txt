0. API仕様

■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■

unl_cpu		CPUロック状態の解除〔T〕
iunl_cpu	CPUロック状態の解除〔I〕

【C言語API】
	ER ercd = unl_cpu()
	ER ercd = iunl_cpu()

【パラメータ】
	なし

【リターンパラメータ】
	ER			ercd		正常終了（E_OK）またはエラーコード

【エラーコード】
	E_CTX			コンテキストエラー（非タスクコンテキストからの呼出
					し：unl_cpuの場合，タスクコンテキストからの呼出し：
					iunl_cpuの場合）
	E_OACV〔P〕		オブジェクトアクセス違反（システム状態に対する通常
					操作2が許可されていない：unl_cpuの場合）

【機能】

CPUロックフラグをクリアし，CPUロック解除状態へ遷移する．CPUロック解除状
態で呼び出した場合には，何も行われずに正常終了する．

マルチプロセッサ対応カーネルにおいて，unl_cpu／iunl_cpuを呼び出したプロ
セッサによって取得されている状態となっているスピンロックがある場合には，
unl_cpu／iunl_cpuによってCPUロック解除状態に遷移しない（何も行われずに
正常終了する）．

【補足説明】

マルチプロセッサ対応カーネルでは，CPUロック解除状態へ遷移した結果，ディ
スパッチ保留状態が解除され，ディスパッチが起こる可能性がある．また，保
護機能対応カーネルとマルチプロセッサ対応カーネルでは，タスク例外処理ルー
チンの実行が開始される可能性がある．

1.2. 正常条件のテストケース(FMP)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
(F-a) 自プロセッサがCPUロック状態で，他プロセッサがCPUロック状態で呼び
      出す場合，自プロセッサのCPUロックフラグをクリアし，CPUロック解除
      状態へ遷移すること．(ASP:b，EX_CARE_STATE)

(F-b) 自プロセッサがCPUロック解除状態で，他プロセッサがCPUロック状態で
      呼び出す場合，何も行われずに正常終了すること．
      (ASP:c，EX_CARE_STATE)

(F-c) 自プロセッサがスピンロック状態で，他プロセッサがCPUロック状態で呼
      び出す場合，何も行われずにCPUロック解除状態に遷移しないこと．
      (ASP:-，EX_NEW_STATE)

(F-d) 自プロセッサがCPUロック状態で，実行状態のタスクより優先順位が高い
      タスクがある場合，CPUロック解除状態へ遷移され最も優先度が高いタス
      クが実行状態になること．(ASP:-，EX_NEW_STATE)

ER
unl_cpu(void)
{
	ER		ercd;

	LOG_UNL_CPU_ENTER();
	CHECK_TSKCTX();

	if (t_sense_lock() && ((get_my_p_pcb())->locspnid == 0)) {
		t_unlock_cpu();
	}
	ercd = E_OK;

  error_exit:
	LOG_UNL_CPU_LEAVE(ercd);
	return(ercd);
}
